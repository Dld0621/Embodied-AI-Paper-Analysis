# 阅读方法论

> 怎么读，比读多少更重要。本文记录一套在具身智能论文上反复验证有效的三遍阅读法，以及把"读过"沉淀成"能用上"的具体动作。

## 三遍阅读法

具身智能论文密度高、跨领域，一次读透既不现实也不必要。把阅读拆成三遍，每遍目标不同、耗时递增，可以在任何一篇论文上按需停在最划算的那一遍。

### 第一遍：速读抓主旨（10-15 分钟）

只看标题、摘要、引言首尾段、所有图表与结论。读完后用一句话回答三个问题：

1. 它解决什么问题？
2. 用了什么类型的方法（学习/优化/几何/系统）？
3. 与我关心的事有没有关系？

如果第三个问题的答案是"否"或"弱相关"，这篇论文到此为止，不必往下。速读的产出是 `summaries/` 里的一句话速览，便于日后检索时快速判断要不要重读。

### 第二遍：精读啃方法（40-90 分钟）

带着第一遍的问题，逐节读方法与实验，重点是：

- 输入输出到底是什么，数据从哪来
- 网络结构或优化目标的**关键假设**——它成立的前提条件，往往是迁移失败的真因
- 主表里的最强基线和消融，判断提升是来自核心创新还是工程调参

这一遍对应模板的"核心方法""关键创新""实验与结果"三节。读的时候随手把公式、流程图抄进笔记，不要依赖记忆。

### 第三遍：批判读找启发（不限时）

只在少数高价值论文上进行。做法是反过来质疑：

- 它的假设在我的场景下是否成立？（例：假设有大规模仿真数据，但我只有真人遥操作数据）
- 哪个模块可以拆出来单独用？
- 如果把它的方法换成我已有的某个组件，会更好还是更差？

这一遍的产出是模板里"与本项目的关联"一节，也是整个仓库最有价值的内容。

## 沉淀动作

读完不等于学会。每篇论文至少触发以下一个动作，否则不算读完：

- **归档**：按模板填完笔记，归入对应主题目录
- **对照**：找一篇同主题旧笔记，比较两者方法差异，在双方笔记里互加交叉链接
- **追问**：把"与本项目的关联"里的待验证想法转成一个具体实验或 Issue
- **复现**：对最关键的 1-2 篇跑通最小复现，把踩坑记入"复现笔记"

## 防止读完即忘的几条纪律

- **先写一句话总结再读正文**。迫使自己带着预测去读，发现偏差时记忆更深
- **图表优先于正文**。具身智能论文的核心信息常浓缩在系统图与主表里，正文往往是论证
- **记假设而非记结论**。结论会过时，假设不会；标注清"在 X 条件下成立"比抄一句结论有用得多
- **定期回扫**。每完成一个主题目录，回看该目录全部笔记的一句话总结，更新主题索引里的判断
